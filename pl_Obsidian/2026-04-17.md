# Modo de Uso

Para entender cómo y en qué orden se ejecutan los módulos del sistema de predicción NBA que hemos construido, es importante ver la arquitectura general del pipeline. El flujo está diseñado para ser **secuencial y orquestado**, donde cada etapa produce datos que alimentan a la siguiente. A continuación te explico el orden de ejecución, los puntos de entrada y cómo interactúan los componentes.

### **1. Punto de Entrada Principal**

El script se ejecuta desde el bloque `if __name__ == "__main__":` al final del archivo del pipeline completo:

```python
if __name__ == "__main__":
    config = {
        'cache_dir': './nba_cache',
        'models_dir': './models',
        'data_dir': './data',
        'seasons': ['2023-24', '2024-25'],
        'confidence_threshold': 0.95,
        'initial_bankroll': 1000,
    }
    
    pipeline = NBAPredictionPipeline(config)
    results = pipeline.run_full_pipeline(skip_ingest=False)
```

### **2. Orden Detallado de Ejecución dentro de `run_full_pipeline()`**

```python
def run_full_pipeline(self, skip_ingest: bool = False) -> Dict:
    # ──────────────────────────────────────────────────────────
    # PASO 1: INGESTA DE DATOS
    # ──────────────────────────────────────────────────────────
    if skip_ingest and raw_file_exists:
        df_raw = pd.read_parquet('data/nba_raw.parquet')
    else:
        df_raw = self.step1_ingest_data()  # <-- Llama a NBADataCollector
    # ──────────────────────────────────────────────────────────
    # PASO 2: FEATURE ENGINEERING
    # ──────────────────────────────────────────────────────────
    df_features = self.step2_engineer_features(df_raw)  # <-- Llama a NBAFeatureEngineer
    # ──────────────────────────────────────────────────────────
    # PASO 3: ENTRENAMIENTO DEL MODELO
    # ──────────────────────────────────────────────────────────
    model_results = self.step3_train_model(df_features)  # <-- Llama a NBAPredictionModel
    # ──────────────────────────────────────────────────────────
    # PASO 4: SIMULACIÓN DE APUESTAS (requiere predicciones)
    # ──────────────────────────────────────────────────────────
    simulation_results = self.step4_simulate_betting(predictions_df, odds_data)
    # ──────────────────────────────────────────────────────────
    # GUARDAR RESULTADOS
    # ──────────────────────────────────────────────────────────
    return results
```

### **3. Ejecución Paso a Paso (Interna de cada método)**

#### **Paso 1 – Ingesta de Datos (`step1_ingest_data`)**

1. **`NBADataCollector.get_game_ids()`**
   - Itera sobre las temporadas especificadas (ej. `['2023-24', '2024-25']`).
   - Para cada temporada, paraleliza la obtención de `GAME_ID` por equipo usando `ThreadPoolExecutor`.
   - Usa `leaguegamefinder.LeagueGameFinder` con `league_id_nullable="00"`.
   - Almacena caché en `./nba_cache/game_ids_2023-24_2024-25.pkl`.

2. **`NBADataCollector.collect_all_boxscores()`**
   - Toma la lista de `game_ids` y los procesa en batches (tamaño por defecto 100).
   - Para cada batch, usa `ThreadPoolExecutor` para llamar a `fetch_boxscore()` (que usa `boxscoretraditionalv3.BoxScoreTraditionalV3`).
   - Implementa **checkpointing**: guarda el progreso en `./nba_cache/boxscores_checkpoint.pkl`.
   - Concatena todos los DataFrames y guarda en `./data/nba_raw.parquet`.

#### **Paso 2 – Ingeniería de Características (`step2_engineer_features`)**

1. **`NBAFeatureEngineer(df_raw)`**
   - Convierte `MIN` a float.
   - Calcula métricas avanzadas (`eFG%`, `TS%`, `POSS`, `OFF_RTG`, etc.).

2. **`prepare_team_level_stats()`** → Agrupación por `GAME_ID` y `TEAM_ID`.

3. **`create_game_level_dataframe()`** → DataFrame con una fila por partido (HOME vs AWAY).

4. **`build_features_dataframe()`** (el corazón del sistema)
   - Itera sobre cada partido en orden cronológico.
   - Para el equipo local y visitante, calcula **promedios móviles** (`last_3_avg`, `last_10_avg`), **EWMA** (`ewma_5`, `ewma_10`), **tendencias** y **desviación estándar**.
   - Calcula versiones condicionadas por localía (`HOME_HOME_*`, `HOME_AWAY_*`, etc.).
   - Añade diferencias entre características de ambos equipos.

5. **`calculate_elo_ratings()`**
   - Simula el rating Elo partido a partido en orden cronológico.
   - Ajusta con ventaja local (70 puntos Elo) y margen de victoria.
   - Devuelve DataFrame con `HOME_ELO`, `AWAY_ELO`, `ELO_DIFF` y se mergea al DataFrame de features.

#### **Paso 3 – Entrenamiento del Modelo (`step3_train_model`)**

1. **`NBAPredictionModel.prepare_data()`**
   - Selecciona columnas de características (excluyendo IDs, fechas, targets).
   - Imputa valores nulos con `KNNImputer`.
   - Escala con `RobustScaler`.
   - Retorna `X, y, dates`.

2. **`time_series_split_evaluation()`**
   - Itera sobre 5 folds de `TimeSeriesSplit`.
   - Evalúa **5 modelos**: Naive Bayes, Logistic Regression, Random Forest, XGBoost, Gradient Boosting.
   - Calcula métricas: accuracy, precision, recall, f1, roc_auc, brier score.
   - Almacena resultados en un diccionario.

3. **`train_best_model()`**
   - Divide datos en 80% train / 20% test **respetando el orden temporal**.
   - Realiza `GridSearchCV` con `TimeSeriesSplit` para optimizar hiperparámetros del modelo elegido (por defecto XGBoost).
   - Calibra probabilidades con `CalibratedClassifierCV` (método isotónico).
   - Evalúa en el conjunto de test y guarda importancia de características.

4. **`analyze_confidence_threshold()`**
   - Calcula precisión para diferentes umbrales de confianza (0.5, 0.55, ..., 0.95).

#### **Paso 4 – Simulación de Apuestas (`step4_simulate_betting`)**

1. **Integración con Odds API** (opcional) para obtener cuotas reales.
2. **`BettingStrategy.evaluate_bet_opportunity()`**
   - Para cada partido con predicción, evalúa si cumple:
     - Confianza ≥ umbral (0.95)
     - Cuotas mínimas (local ≥1.72, visitante ≥2.06)
     - Valor Esperado (EV) positivo (≥2%)
   - Calcula stake usando **Kelly Criterion fraccional**.
3. **`place_bet()`** → Actualiza bankroll y registra la apuesta.
4. **`get_performance_metrics()`** → Calcula ROI, win rate, profit total.

## 🔄 ¿Cómo se Ejecutan los Módulos de Forma Independiente?

Si deseas ejecutar solo una parte del pipeline (por ejemplo, solo ingesta o solo entrenamiento), cada clase puede usarse de forma aislada:

### **Ejemplo 1: Solo Ingesta de Datos**
```python
collector = NBADataCollector(cache_dir='./nba_cache')
game_ids = collector.get_game_ids(seasons=['2024-25'])
df_raw = collector.collect_all_boxscores(game_ids)
df_raw.to_parquet('mis_datos.parquet')
```

### **Ejemplo 2: Solo Feature Engineering con Datos Existentes**
```python
df_raw = pd.read_parquet('data/nba_raw.parquet')
engineer = NBAFeatureEngineer(df_raw)
engineer.prepare_team_level_stats()
engineer.create_game_level_dataframe()
df_features = engineer.build_features_dataframe()
df_elo = engineer.calculate_elo_ratings()
```

### **Ejemplo 3: Solo Entrenamiento con Features Precalculadas**
```python
df_features = pd.read_parquet('data/features.parquet')
model = NBAPredictionModel()
X, y, dates = model.prepare_data(df_features)
results = model.time_series_split_evaluation(X, y, dates)
test_metrics, _, _, _, _ = model.train_best_model(X, y, dates, model_name='XGBoost')
```

## ⏱️ Tiempo de Ejecución Estimado

| Paso                           | Tiempo (para 2 temporadas, ~2500 partidos) |
|--------------------------------|--------------------------------------------|
| Ingesta (con paralelización 8 workers) | 15-25 minutos |
| Feature Engineering            | 3-5 minutos   |
| Entrenamiento + GridSearch     | 2-5 minutos   |
| Simulación                     | < 1 segundo   |

*Nota: El cuello de botella es la ingesta desde la API de la NBA (rate‑limiting implícito). El checkpointing permite reanudar si se interrumpe.*

## 🧠 Conclusión

El sistema está diseñado como un **pipeline orquestado secuencial** pero con componentes **modulares e independientes**. El orden de ejecución es fijo porque cada etapa depende de la salida de la anterior:

**Ingesta → Features → Modelo → Apuestas**

Esta arquitectura permite:
- Ejecutar el flujo completo con `run_full_pipeline()`.
- Ejecutar etapas individuales para depuración o reentrenamiento.
- Persistir resultados intermedios para evitar recalcular.
