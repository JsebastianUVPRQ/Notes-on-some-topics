You're describing a common Windows problem where, during a restart (or shutdown), you get a dialog box saying something like "Close these programs before restart" or "This app is preventing you from restarting," but the app has no name—just a generic icon/logo (like a blank square or default .exe icon). This makes it hard to spot in Task Manager. It's usually not malware, but often a glitchy background process, startup app, or system feature that doesn't close cleanly. I'll walk you through safe, step-by-step fixes for Windows 10/11.

### Quick First Steps
1. **Force Close via the Dialog**: When the message appears, click "Shut down anyway" or "Restart anyway" (it might take 10-30 seconds, but it's safe—no data loss risk for most apps).
2. **Manual Check Before Restarting**:
   - Press **Ctrl + Shift + Esc** to open Task Manager.
   - Go to the **Processes** tab (or **Details** for more info).
   - Look for suspicious processes with high CPU usage or no description (sort by "Status" for "Not responding").
   - Right-click any odd one > **End task**. Avoid ending system processes like explorer.exe unless you're sure.
   - If nothing shows, try the fixes below.

### Fix 1: Identify the Culprit Using Event Viewer (Easiest Way to Name It)
This logs exactly which process is blocking the restart, even if it's nameless in the dialog.

1. Right-click the Start button > **Event Viewer**.
2. In the left pane, expand **Windows Logs** > **Application**.
3. In the right pane, click **Filter Current Log...**.
4. Under **Event sources**, select **Application**, and in **Event ID**, enter `1001`.
5. Click **OK**, then look for recent events (sort by date) with a description like "The following application attempted to veto the shutdown" or "Windows was prevented from shutting down."
6. Double-click the event—the app's .exe name (e.g., "steam.exe") will be listed after the colon. Google the name to identify it (e.g., if it's "G.exe," it's often a Google-related process).
7. Once identified, close that app manually before future restarts, or uninstall it via **Settings > Apps > Apps & features**.

If no events show, proceed to the next fixes.

### Fix 2: Disable Fast Startup (Common Cause)
Fast Startup hibernates apps instead of fully closing them, leading to nameless blockers.

1. Search for **Power & sleep settings** in the Start menu > Click **Additional power settings** (on the right).
2. Click **Choose what the power buttons do** (left sidebar).
3. Click **Change settings that are currently unavailable** (top).
4. Uncheck **Turn on fast startup (recommended)**.
5. Click **Save changes**, then restart to test.

### Fix 3: Perform a Clean Boot (Pinpoint Startup Apps/Services)
This temporarily disables non-Microsoft startup items to isolate the problem.

1. Press **Win + R**, type `msconfig`, and hit Enter.
2. Go to the **Services** tab > Check **Hide all Microsoft services** > Click **Disable all**.
3. Go to the **Startup** tab > Click **Open Task Manager**.
4. In Task Manager's Startup tab, right-click each item > **Disable** (except Microsoft ones).
5. Back in msconfig, click **OK** > Restart.
6. Test a restart—if it works, re-enable items one by one (via msconfig or Task Manager) and restart each time to find the guilty app (e.g., Steam, Docker, or antivirus).

To undo: Run msconfig again > **Normal startup** > OK > Restart.

### Fix 4: Auto-Force Close Apps on Shutdown (Registry Tweak)
This makes Windows automatically end hung apps without prompting (use if you're okay skipping save prompts).

1. Press **Win + R**, type `regedit`, and hit Enter (approve UAC).
2. Navigate to: `HKEY_CURRENT_USER\Control Panel\Desktop`.
3. Right-click the right pane > **New > String Value** > Name it `AutoEndTasks`.
4. Double-click it > Set **Value data** to `1` > OK.
5. Close Registry Editor > Restart to apply.

To reverse: Delete the `AutoEndTasks` value or set it to `0`.

### Fix 5: Run System File Checks (If Corrupted Files Are Involved)
1. Search for **cmd** > Right-click **Command Prompt** > **Run as administrator**.
2. Type these one by one, pressing Enter after each:
   ```
   sfc /scannow
   ```
   (Wait for it to finish—repairs system files.)
   ```
   DISM /Online /Cleanup-Image /RestoreHealth
   ```
   (Fixes Windows image if needed.)
3. Restart and test.

### Other Tips
- **Update Windows/Apps**: Go to **Settings > Update & Security > Windows Update > Check for updates**. Outdated apps (e.g., after a Windows update) often cause this.
- **Scan for Malware**: Run **Windows Security** (search for it) > **Virus & threat protection > Scan options > Full scan**. Or use free tools like Malwarebytes.
- **If It's a Specific App**: Common culprits include Steam (close it fully), Free Download Manager, Logi Options+ (Logitech software), or Docker. Uninstall via Settings if identified.
- **Prevent Reopening Apps**: If apps reopen after restart (related issue), go to **Settings > Accounts > Sign-in options** > Turn off **Automatically save my restartable apps...**.
